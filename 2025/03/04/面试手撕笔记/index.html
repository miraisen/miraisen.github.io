在栈的实现类选取上：

Stack：Java早期提供的用于实现栈数据结构的类，继承自Vector类。Vector是线程安全的动态数组，意味着Stack也是线程安全的，但其线程安全是通过在方法上加synchronized关键字实现的，在单线程环境下会带来额外的性能开销。

八股：Vector和Stack的区别？

Deque：是一个接口，即双端队列(Double - Ended Queue)，继承自Queue接口。Deque接口定义了可以在队列两端进行插入、删除和访问元素的方法，既可以当作栈使用，也可以当作队列使用。

ArrayDeque：是Deque接口的一个具体实现类，基于数组实现，内部使用循环数组来存储元素。实现了Deque接口的所有方法，具备双端队列的功能，随机访问性能较好，没有容量限制（自动扩容）。

在 Java 中，可以使用 Deque 接口当作栈来使用，Deque 提供了栈操作所需的方法，如 push()、pop() 和 peek() 等。通常会选择 ArrayDeque 或 LinkedList 作为 Deque 接口的具体实现类。

空栈处理：

  • Stack抛出EmptyStackException异常。
  • ArrayDeque：当调用removeFirst()或peekFirst()方法时，如果队列为空，removeFirst()会抛出NoSuchElementException异常，peekFirst()会返回null。还提供了pollFirst()方法，当队列为空时返回null，不会抛出异常。

总结：

  • 如果需要在多线程环境下使用栈，并且对线程安全有严格要求，可以使用 Stack 类。

  • 在单线程环境下，推荐使用 ArrayDeque 来实现栈的功能，因为它性能更好，并且提供了丰富的操作方法。

  • 当需要当作栈使用的 Deque 时，推荐优先使用 ArrayDeque，因为它通常在性能上更具优势。

  • 若需要频繁进行插入和删除操作，且可能涉及到链表操作的特性（如随机访问等），可以考虑使用 LinkedList。

对于本题，选择Stack和ArrayDeque的处理不一样。

73.矩阵置零

要求是仅使用常量空间。使用第一列作为该行是否需要置零的标识。先遍历第一列，如果第一列有零，表示第一列也需要置零，但是先不要置零，因为我们要使用第一列作为标识。

然后置零是判断这一行或者这一列有零就把该元素置为零。要注意的是使用倒序遍历，避免修改标记信息。

通过从矩阵的右下角开始逐行逐列地遍历，我们可以确保在修改当前元素时，不会影响到后续元素的判断依据。因为我们是从后往前处理，所以在处理当前元素时，其对应的第一行和第一列的标记信息仍然是原始的标记信息，没有被后续的修改所破坏。

回溯

解决组合优化、搜索等问题。通过深度优先搜索的方式来遍历问题的解空间数，以寻找满足特定条件的解或最优解的算法。

原理：利用深度优先搜索策略，从问题的初始状态出发，递归地尝试所有可能的选择，在每一步选择时，判断当前状态是否满足问题的约束条件。如果满足，则继续递归探索下一个状态；如果不满足，则回溯到上一个状态，尝试其他选择。

  1. 定义解空间：确定问题的所有可能解的集合，通常可以用一个树形结构来表示，称为解空间树。例如，对于一个字符串的全排列问题，解空间树的根节点表示空字符串，每个分支表示在当前字符串后面添加一个字符，叶子节点就是所有的全排列结果。
  2. 确定约束条件：明确问题的约束条件，用于判断当前状态是否合法。比如在八皇后问题中，约束条件是任意两个皇后不能在同一行、同一列或同一斜线上。
  3. 深度优先搜索：从根节点开始，按照深度优先的顺序遍历解空间树。在每个节点处，根据约束条件判断是否可以继续扩展当前节点。如果可以，则选择一个未尝试过的子节点进行递归搜索；如果不可以，则回溯到父节点，尝试其他子节点。
  4. 记录解：当找到一个满足问题要求的解时，将其记录下来。可以根据具体问题的要求，决定是继续寻找其他解还是直接结束算法。

222.完全二叉树的节点个数

规定根节点位于第0层，完全二叉树最左边的节点一定位于最底层，因此从根节点出发，每次访问左子节点，直到遇到叶子节点，该叶子节点即为完全二叉树的最左边的节点，经过的路径长度就是最大层数h。

对于最大层数为h的完全二叉树，节点个数一定在 $[2^h, 2^(h+1)-1]$的范围内。

可以在该范围内通过二分查找的方式得到完全二叉树的节点个数。

计算$2^h$最快的方法是移位计算，(1 << h)

好理解一点的方法：

对左右子树进行高度统计，记为left和right，有两种情况：left == right、left != right。分别表示

  • 左子树是满二叉树

    左子树的节点总数等于 2^left - 1 加上root节点，刚好是2 ^ left。再对右子树进行递归统计。

  • 最后一层不满，倒数第二层已经满了

    可以直接得到右子树的节点个数，同理是2^right。再对左子树进行递归查找。

完全背包问题

1.求背包至少能装多大价值的物品

标准的完全背包问题，使用动规解决。定义状态dp[i][j]表示前i个物品放入容量为j的背包中所能获得的最大价值。状态转移方程为：

  • 当第i个物品的重量weights[i-1]大于当前背包容量j时，不能放入该物品，dp[i][j] = dp[i-1][j]

  • 当第i个物品的重量weights[i-1]小于等于当前背包容量j时，可以选择放入或不放入该物品

    Math.max(dp[i][j], dp[i][j-weights[i-1]] + valuess[i - 1]);

2.若背包恰好装满，求至多能装多大价值的物品

仍然设 dp[i][j]表示前 i 种物品放入容量为 j 的背包中所能获得的最大价值。

将 dp[0][0]初始化为0， 表示没有物品放入容量为0的背包时价值为0，对于j > 0的dp[0][j]，初始化为一个极小值，因为没有物品时无法装满背包，所以这些状态是无效的。

状态转移方程与标准完全背包问题的状态转移方程相同。

完全背包问题和0-1背包问题的区别：

0-1背包问题：每种物品只有一个，对于每个物品只有选或不选两种情况。状态转移时，dp[i][j]由dp[i-1][j]（不选第i个物品）和 dp[i-1][j-weights[i-1]]+values[i-1]（选第i个物品）推导而来，因为物品只有一个，选了它就不能再选，所以是从 dp[i-1]这一行的状态转移过来。

完全背包问题：每种物品有无限个。在状态转移时，当第i个物品的重量小于等于当前背包容量 j 时，虽然同样考虑放入或不放入该物品， 但由于物品可以多次选取，放入物品时是从 dp[i][j-weights[i-1]]+values[i-1]推导，也就是从当前行dp[i]的状态转移过来，意味着可以重复选择第 i 个物品，而不像0 - 1背包那样只能选一次。

Kadane算法

101001001，翻转一个任意长度的子串，使1的数量最多，求最多的1的数量。

可以通过将问题转化为最大子数组和问题来优化时间复杂度。把 0 看作 1，1 看作 -1，那么翻转一个子串就相当于将该子串对应的数值取反，目标是让所有数值的和最大。这可以使用 Kadane 算法来解决，时间复杂度可以优化到 O(n)。

最大子数组和：

Kadane 算法（卡丹算法）是一种用于解决最大子数组和问题的高效算法，由美国计算机科学家 Jay Kadane 设计，其时间复杂度为 (O(n))，其中 n 是数组的长度。

Kadane 算法的核心思想是使用动态规划的思想，通过一次遍历数组，在遍历过程中维护两个变量：

  • 当前最大子数组和：表示以当前元素结尾的连续子数组的最大和。

  • 全局最大子数组和：表示整个数组中连续子数组的最大和。

    在遍历数组时，对于每个元素，有两种选择：将当前元素加入到之前的子数组中 / 以当前元素作为新的子数组的起始。
    选择哪种方式取决于哪种方式能得到更大的和。

122.买卖股票的最佳时机II

状态数组：f[i][0]表示第i天结束时不持有股票的最大利润，f[i][1]表示第i天结束时持有股票的最大利润。数组长度为i+1是为了方便处理第0天的初始状态。

第0天结束时不持有股票的最大利润初始化为0，第0天结束时持有股票是不可能的情况，初始化为 Integer.MIN_VALUE，表示这种情况是不合法的。

状态转移方程：

f[i + 1][0] 表示第 i + 1 天结束时不持有股票的最大利润，有两种情况可以转移到这个状态：
第 i 天就不持有股票，第 i + 1 天保持不持有股票的状态，即 f[i][0]。
第 i 天持有股票，第 i + 1 天卖出股票，获得 prices[i] 的利润，即 f[i][1] + prices[i]。

最终的最大利润是第 n 天结束时不持有股票的最大利润，因为最后一天持有股票是没有意义的，所以返回 f[n][0]。

可以将空间复杂度优化到 O(1)，因为状态转移只依赖于前一天的状态，不需要保存整个状态数组。

快速排序

快速排序的核心思想是通过选择一个基准元素（pivot），将数组分为两部分，使得左边部分的所有元素都小于等于基准元素，右边部分的所有元素都大于等于基准元素，然后分别对左右两部分递归地进行快速排序，最终得到一个有序的数组。

算法步骤：选择基准元素 - 分区操作 - 递归排序
